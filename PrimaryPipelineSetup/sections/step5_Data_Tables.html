Data Tables

File:
  • Files that contain putative cells only:
    • [sample_name]_RSEC_MolsPerCell_MEX.zip
    • [sample_name]_DBEC_MolsPerCell_MEX.zip

  • Files that contain putative cells and unfiltered cells:
    • [sample_name]_RSEC_MolsPerCell_Unfiltered_MEX.zip

The data table output files containing information for putative cells as well as the unfiltered cells in MEX format. MEX format is an efficient way to store sparse dataset and is a common input format for many single cell analysis tools. Users with experience in secondary analysis may use this file as an alternative to the seurat object or Python object in secondary analysis.

The MEX.zip files contain three separate gzip compressed files that together represent the molecule counts of each bioproduct in each cell. By convention, these files are named:

  1. Barcode.tsv:

    Containing a list of cells indexed (integer between 1 to 384^3), one per row.
    They barcodes are sorted numerically.

  2. Features.tsv:

    Containing a list of bioproducts detected, one per row.
    The bioproducts are sorted alphabatically.
    For improved compatibility, this file contains three columns:

    • First 2 columns: a duplicated gene symbol or AbSeq
    • Thisrd column: Indicates mRNA (Gene Expression) or AbSeq (Antibody Capture) types.

  3. Matrix.mtx:

    Containing a three column per row representation of molecule counts.

    • First column is the 1-based row number from features.tsv (bioproduct).
    • Second column is the 1-based row number from barcodes.tsv (cell).
    • Third column is the molecule count detected for that bioproduct in that cell.
