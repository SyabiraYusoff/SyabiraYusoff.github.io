Pipeline Output Files Explained

General Output Files

These files are generated by the pipeline and contain important information about the analysis.

Bioproduct Satistics:

The molecular identifier adjustment algorithms RSEC and DBEC are applied to each bioproduct. The molecular identifier metrics file lists the metrics from RSEC and DBEC on a per-bioproduct basis. For more information on RSEC and DBEC molecular identifier adjustment algorithms, see Molecules and error correction.

Immune Cell Classification

File: [sample_name]_(assay)_cell_type_experimental.csv

These files contain the immune cell classification results. The classification is based on the expression of specific genes and is used to identify different immune cell types. We collected a large dataset of PMBCs that had targeted immune response panel (397 genes) and AbSeq expression. Performed dimensionality reduction. We used following cell types for classification:

  • Classical monocyte
  • Nonclassical monocyte
  • Dendritic cell
  • B cell
  • Naive CD4 T cell
  • Memory CD4 T cell
  • Naive CD8 T cell
  • Memory CD8 T cell
  • Natural killer cell
  • Gamma-delta T cell

If customer using other than PBMCs, it is recommended to apply your own cell type classification in the downstream analysis.

When the Immune Cell Classification is done:

  • Using WTA/AbSeq cell calling data: [sample_name]_cell_type_experimental.csv
  • using ATAC cell calling data: [sample_name]_ATAC_cell_type_experimental.csv
  • using WTA/AbSeq and ATAC cell calling data: [sample_name]_Joint_cell_type_experimental.csv


BAM and BAM Index

File:
  • [sample_name]_Bioproduct.bam
  • [sample_name]_Bioproduct.bam.(bai|csi)
  • [sample_name]_ATAC.bam
  • [sample_name]_ATAC.bam.(bai|csi)

    Note:

  1. BAM file will be available when the setting is activated prior running the pipeline. User need to re-run the pipeline if the BAM is needed after pipeline ran.
  2. The *_Bioproduct.bam consists of reads arising from AbSeq, SampleTag, Targeted, VDJ or WTA assays.
  3. BAM Index is the index file associated with the coordinate-sorted BAM file.
  4. A Combined_ prefix is added to the bam and bai/csi files when SampleTags are present in the experiment to signify that the resulting bam contains reads from all the samples.
Further information:

  • BAM is an alignment file in binary format that is generated by the aligner and contains tags related to alignment quality.
  • Bioproduct BAM consists of alignments from the R2 reads only, whereas the ATAC BAM contains alignments from both the R1 and R2 reads.
  • BAM files are sorted according to the alignment coordinates of either the R2 read (Bioproduct) or both the R1 and R2 reads (ATAC) on each chromosome.
  • BAI index is typically generated; however, if any contig in the reference genome exceeds 500 Mb in length, a CSI index is created instead, as it supports larger contig sizes beyond the BAI index specifications.
  • The BD Rhapsody™ Sequence Analysis Pipeline further annotates the BAM files with the tags description below:

For the Bioproduct BAM, if a read has multiple alignments (NH tag > 1), then only the first alignment (HI tag is 0 or 1) will be annotated with tags 'MA' and 'CN'.

Tag Definition
NH Number of reported alignments that contain the query in the current record.
HI Query hit index.
AS Alignment score assigned by the aligner.
NM Edit distance to the reference.
CB A number between 1 and 3843 representing a unique cell label sequence (CB = 0 when no cell label sequence is detected).
MR Raw molecular identifier sequence. (Bioproduct BAM only)
MA RSEC-adjusted molecular identifier sequence. If not a true cell, the raw UMI is repeated in this tag. (Bioproduct BAM only)
CN Indicates if a sequence is derived from a putative cell, as determined by the cell label filtering algorithm (T: putative cell; x: invalid cell label or noise cell).
ST The value is 1–24, indicating the Sample Tag of the called putative cell, or M for multiplet, or x for undetermined.
XF Name of the Gene/AbSeq/SampleTag that a particular read was annotated to. (Bioproduct BAM only)
MQ Mapping quality of the mate/next segment
MC CIGAR string for mate/next segment
MD Mismatching positions/bases
XS Suboptimal alignment score
ms The ms tag is produced samtools fixmate, and can be used by samtools markdup and other tools to identify duplicate reads.

Data Tables

File:
  • Files that contain putative cells only:
    • [sample_name]_RSEC_MolsPerCell_MEX.zip
    • [sample_name]_DBEC_MolsPerCell_MEX.zip

  • Files that contain putative cells and unfiltered cells:
    • [sample_name]_RSEC_MolsPerCell_Unfiltered_MEX.zip

The data table output files containing information for putative cells as well as the unfiltered cells in MEX format. MEX format is an efficient way to store sparse dataset and is a common input format for many single cell analysis tools. Users with experience in secondary analysis may use this file as an alternative to the seurat object or Python object in secondary analysis.

The MEX.zip files contain three separate gzip compressed files that together represent the molecule counts of each bioproduct in each cell. By convention, these files are named:

  1. Barcode.tsv:

    Containing a list of cells indexed (integer between 1 to 384^3), one per row.
    They barcodes are sorted numerically.
    Click here to download Cell label structure, cell label sequences, bead sequences, and python utility functions.
    To download the whitelist in fasta format:

    • Original V1 beads:Rhapsody_cellBarcodeV1_IndexToSequence.fasta.zip
    • Enhanced beads:Rhapsody_cellBarcodeEnh_IndexToSequence.fasta.zip
    • Enhanced V2/V3 beads:Rhapsody_cellBarcodeEnhV2_IndexToSequence.fasta.zip

  2. Features.tsv:

    Containing a list of bioproducts detected, one per row.
    The bioproducts are sorted alphabatically.
    For improved compatibility, this file contains three columns:

    • First 2 columns: a duplicated gene symbol or AbSeq
    • Thisrd column: Indicates mRNA (Gene Expression) or AbSeq (Antibody Capture) types.

  3. Matrix.mtx:

    Containing a three column per row representation of molecule counts.

    • First column is the 1-based row number from features.tsv (bioproduct).
    • Second column is the 1-based row number from barcodes.tsv (cell).
    • Third column is the molecule count detected for that bioproduct in that cell.
